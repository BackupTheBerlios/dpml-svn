Ant tasks that provide the entry point to transit based repository access for build system. The init task is used to established a number of transit related ant properties and the plugin task is used to load antlibs, taskdefs, typedefs and build listeners using classloaders established relative to resources managed by the transit system.

Example build file:

<project name="example" xmlns:transit="antlib:net.dpml.transit">

  <transit:import uri="local:template:magic/reactor"/>

</project>

Example of a request to load and register a task definition.

<project name="example" xmlns:transit="antlib:net.dpml.transit">

    <transit:plugin uri="artifact:plugin:dpml/magic/dpml-magic-doc#3000">
      <task class="net.dpml.magic.doc.DocTask" name="docs"/>
    </transit:plugin>

    <docs/>

</project>
